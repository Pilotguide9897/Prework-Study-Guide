Terminal/Command Line

  • The command line allows an individual to input text commands to execute operating system functions (e.g., creating a new folder or opening a file).
  • The terminal is the application that allows people to interact with the shell, the software that interprets and executes the various commands that get typed into the terminal.

    • The following are some commands that may be useful when learning to use the terminal:

  • cd: This command allows us to change directories.
  • ls: This command allows us to see a list of files and directories on our computer.
  • pwd: This command is short for 'print working directory,' this command prints out the directory that we are currently in inside the terminal.
  • cd .. :This command allows you to navigate backward one directory at a time.
  • mkdir: This command allows you to "MaKe a directory. To create a named directory, simply type mkdir (name of directory to be created).
  • touch: This command allows the creation of a new file. You write touch (name of file).hypothetical file type.
  • open: This command is used to open files in the macOS command line. The start command is used for Windows (e.g., start (name of file).hypothetical file type).
  • code: The code command is used if you want to open a file using VS Code (e.g. code index.html). You can also open multiple files at once if they are in the same location in a file directory (e.g., code index.html style.css documentation\readme.md).
  • mv: The mv command may be used to move files or folders from one location to another on the same computer. Note that the mv command moves the file or folder from its old location and puts it in the new location, it does not create a copy. For more information on how to copy a file see below. The mv command also allows you to rename your file/folder and also change the location of the renamed file if you wish to do so (e.g., mv old-folder-name new-folder-name). One thing that you can do is type mv and then open finder and drag the file onto the terminal to copy the full path of the file to the terminal.
  • cp: The cp command is used to make a copy of a file. For example, to copy a folder named Expenses in your Documents folder to another volume named Data: (% cp -R ~/Documents/Expenses /Volumes/Data/Expenses). The -R flag causes cp to copy the folder and its contents. Note that the folder name does not end with a slash, which would change how cp copies the folder.

SSH Keys

  • An SSH key is an access credential for the secure shell (SSH) network protocol.
  • Generating SSH keys allows developers to interface with certain remote services without having to constantly retype login information.
  • The following is a summary of the steps to check for existing SSH keys: (1) Enter ls -al ~/.ssh in your terminal (Git Bash for Windows or Terminal for macOS) to see if any existing SSH keys are present. (2) Look for a public SSH key that ends in .pub. For example, id_rsa.pub, id_ecdsa.pub, or id_ed25519.pub. If you see an existing SSH key pair (a public SSH key like id_rsa.pub and a private SSH key like id_rsa), then you can skip generating a new SSH key and use your existing keys to add to the ssh-agent in a later step. If you receive an error that it does not exist, that means you will need to generate a new SSH key pair.
  • To generate an SSH key: Enter ssh-keygen -t ed25519 -C “your_email@example.com” in your terminal to create a new SSH key. When prompted, save your key in the default file location and create a secure passphrase.
  • Once you have your SSH key, you can add them to the ssh-agent by: (1) Entering eval “$(ssh-agent -s)” in your terminal to start the ssh-agent. For macOS, you might need to run a few additional commands to start the ssh-agent and add your SSH key. That can include running sudo -s -H before starting the ssh-agent, or you may need to use exec ssh-agent bash or exec ssh-agent zsh to run the ssh-agent. Follow the GitHub Docs carefully to do this step successfully. Add your private SSH key to the ssh-agent using the ssh-add command.

HTML

  • HTML is used to create the structure of a webpage. You can think of HTML as the frame of a website. It is made up of elements that work together to provide content and the basic framework for all pages on the web.
  • Elements are a type of content on a webpage. Common types of elements in HTML are: (1) Text elements, (2) Image elements, (3) Header elements, and (4) List elements. These elements add visible information on the page.
  • In addition to visible elements, there are also elements that are used to divide the content on a page and describe the content to browsers and other developers. Examples of these are: (1) Meta-related elements, (2) Content division elements, and (3) Semantic elements.
  • In HTML, elements are composed of an opening tag and a closing tag represented with angled brackets [< >]. These tags contain everything that we want to include about the element and include content and even other nested elements.
  • Each HTML element performs a unique role. Two especially important elements, however, are the head and body elements.
  • The head element contains information about the webpage that helps the browser render the page correctly.
  • The body element represents the visible content shown to the user. All elements that we want the user to be able to see and interact with must be placed inside the element.
  • You use the h(1-6) tag to add a header of certain levels.
  • You can use the ul tag to add a list. Each element in the list needs to be indicated by the li element.
  • You need to use multiple li elements to add more items. Each one represents a new bullet point in your list.
  • The class attribute is used to point to a class name in a style sheet and can also be used by JS to access and manupulate elements with the specific class name.

CSS

  • CSS is useful for adding visual elements to websites.
  • A margin indicates how much space we want around the outside of an element.
  • A padding is not quite the same as a margin, although people often confuse them. A padding indicates how much space we want around the content inside of an element.
  • We can assign colours in css using semantic reference such as, grey, or using a hexidecimal code (hex #) to refer to a colour.
  • The border property assigns a border.
  • A period (.) in front of a selector designates that as a class selector, meaning that any HTML element assigned to the class will gain the specified propery assignments from that block of CSS.
  • There are multiple ways to incorporate CSS into your HTML document: (1) Inline - by using the style attribute inside an HTML element, (2) Internal - by using a style element in the head section, and (3) External - by using a link element to link to an external css file. The link element goes in the head element of the website because that is where the metadata of the webpage is stored.

Git

  • git status: Checks what branch we are currently on.
  • git branch: Lists your branches. An *will appear next to the currently active branch.
  • git checkout: -b branch-name: Creates a new branch and switches to it.
  • git branch [branch-name]: Creates a new branch at the current commit.
  • git merge [branch]: Merge a specified branch's history into the current one.
  • git fetch [alias]: Fetch down all the branches into your current branch to bring it up to date.
  • git push [alias]/[branch]: Transmit local branch commits to the remote repository branch.
  • git pull: Fetch and merge any commits from the tracking remote branch.
  • git add -A: used to commit all the files in the working branch.
  • git commit -m "Insert your comment here": You use the -m tag to associate a message with your commit. Commit messages should be short, descriptive messages of the changes implemented.
  • git pull origin main: Used to pull the base branch. It is a good habit to always pull the base branch into your feature branch before you push any code up.
  • Version control is an important part of the web development process.

Javascript

  • JavaScript is used to add functionality and interactivity to a website.
  • A variable is a named container that allow us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are the building blocks of JavaScript. When we create a function, we are defining a block of code that performs a specific task, and giving it a unique name. However, just defining a function does not mean that it will execute in our code automatically. To execute a function, we must call it in our code.
  • We use an open curly bracket after a function to show that it is not adding an arugment.
  • If statements allow us to perform logical comparisons between a value and what you expect. Then it allows you to use that information to perform an action.
  • For loops can be very useful for controlling the work flow. For loops function by running a section of code repeatedly until a certain condition has been satisfied. For loops contain two sections: a header and a body. In addition, (1) The keyword For that starts the loop, (2) the condition being tested, and (3) the EndFor keyword that terminates the loop (For x = Start to End/...Statements to be performed go here/EndFor). The value of x must be an integer variable and the values of Start and End must be integers, integer variables, or integer constants. The major difference between the While and For looping structures is that a For loop will run a predetermined number of times.
  • Since a loop continues until a condition becomes false, be careful not to set up a loop that never becomes false and runs infinitely. This can lock up a computer. You must think carefully about the condition meant to terminate the loop to be sure it becomes false at some time.
  • You use the While-EndWhile statement structure to perform Looping. A While Loop repeats or loops WHILE a condition is true. A While loop consists of three parts: (1) The While key word that begins the loop, (2) the condition to be tested each time the loop iterates or is performed, (3) the EndWhile keyword that ends the loop.
  • JavaScript contains five basic (primitive) data types. These types are strings, numbers, booleans, undefined, and null.
  • Strings are collections of alphaneumeric characters and symbols. Strings are useful for storing letters and words.
  • Booleans can only have two values, true and false. They represent all data which has only two states.
  • Undefined means that a variable has been created, but that it has not been assigned a value.
  • Two other common data types in JavaScript are objects and arrays. Both of these data types consist of collections, and are a way to group together multiple points of data into a single piece.
  • Arrays are a special variable that holds more than one value. Arrays allow you to hold more than one piece of data under a single title and you can access specific numbers in the annex by referring to an index number. It is common practice to declare an array by using the const keyword.
  • When creating an array in JS, spaces and line breaks are unimportant. A declaration may span multiple lines.
  • Objects in JS can be very diverse and may refer to documents, other webpages, etc.
